How Token Vault Works
When an agent needs to call Gmail, it performs a Token Exchange (RFC 8693) with Auth0. Auth0 returns a short-lived, scoped access token from the vault — your refresh tokens never leave Auth0 infrastructure.

Token Revocation Demo

Destructive

Revoking a connection deletes the Auth0 grant via Management API (DELETE /api/v2/grants/{id}). After revocation, any agent call using this connection will immediately fail with a 401 error — demonstrating that the Token Vault enforces authorization boundaries.
